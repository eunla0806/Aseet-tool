# 🧸 Dot Asset Tool: 하이브리드 도트 디자인 보정 전략 (HYBRID_DESIGN_STRATEGY)

본 문서는 외부 생성 AI 이미지(Midjourney, Gemini 등)를 아바타용 도트 스프라이트 아트로 가공할 때 발생하는 3대 치명적 물리적 한계(**디자인 일관성 무너짐, 축소 시 테두리 깨짐, 바디 정렬 불일치**)를 분석하고, 이를 컴퓨터 비전 알고리즘 및 AI 구속 모델로 해결하기 위한 **"하이브리드 보정 파이프라인(Option C)"** 기술 스펙 명세서입니다.

---

## 1. 3대 한계점 및 기술적 원인 분석

외부 AI 이미지 생성 서비스가 제공하는 세련된 고해상도 디자인 초안을 그대로 잘라서 `43 × 68` px 크기의 도트 에셋으로 사용할 수 없는 명확한 과학적 한계는 다음과 같습니다.

### 1.1 디자인 일관성 파괴 (Jittering)
*   **원인**: 외부 생성 AI는 애니메이션의 물리적 흐름(시간에 따른 프레임 변화)을 인지하지 못합니다. 2x2 그리드 내에서 4개의 자세를 한 번에 그리더라도 단추의 개수, 주름의 개수, 옷깃의 각도가 미세하게 다르게 그려지며, 이를 시퀀스로 재생하면 옷이 미친 듯이 떨리는 **지터링(Jittering) 결함**이 발생합니다.

### 1.2 축소 시 테두리 무너짐 (Anti-Aliasing Artifacts)
*   **원인**: 고해상도 초안 이미지는 외곽선과 경계에 수많은 중간 톤 색상(Anti-Aliasing)이 포함되어 있습니다. 이를 무이차 보간인 **Nearest Neighbor로 축소하면 선이 끊기고 면이 톱니바퀴처럼 뾰족하게 깨지는 현상(Aliasing)**이 발생하며, **Bilinear/Lanczos 등으로 축소하면 테두리가 번져 흐리멍덩한 회색빛 노이즈**로 오염됩니다.

### 1.3 아바타 체형/위치 왜곡 (Body-Shape Drift)
*   **원인**: 메이플스토리 월드 아바타 바디 템플릿은 1픽셀 단위의 정확한 인체 비례(어깨선 위치, 목선 깊이, 골반 너비 등)를 요구합니다. 하지만 외부 AI는 인체 구조를 가변적으로 해석(예: 8등신, 숏다리, 긴 팔 등)하여 그리기 때문에, 마우스 오프셋 정렬만으로는 바디 경계를 절대로 완벽하게 맞출 수 없으며 삐져나오거나 비는 왜곡이 생깁니다.

## 2. 해결책: 하이브리드 보정 파이프라인 (Option C)

위 3가지 물리적 한계를 완전히 극복하기 위해, **외부 AI 초안을 "도트의 직접적인 원판"이 아니라 "스타일과 장식의 가이드(Aesthetic Style Guide)"로써 활용**하고, 최종 스프라이트는 **정형화된 규격 바디 위에서 컴퓨터 비전 필터와 ComfyUI를 통해 재구성(Reconstruction)**하는 차세대 아키텍처를 도입합니다.

```txt
┌──────────────────────┐
│ 외부 AI 고해상도 초안 │ ➔ 오직 디자인의 컨셉, 색감, 실루엣 테마만 획득 (위치 정밀도 무시)
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  리샘플링 & 양자화    │ ➔ build_anchor_from_draft.py 에서 강제 색상 수 제한 및 외곽선 선명화 필터 적용
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│ 마스크 클리핑 랩핑   │ ➔ clothing_mask 영역 외부로 삐져나간 바디 정렬 불일치 픽셀 강제 오려내기
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  ComfyUI 동작 확장   │ ➔ 바디 포즈(ControlNet)와 마스크(Inpainting)를 1.0 가중치로 강하게 구속하여 생성
└──────────────────────┘
```

## 3. 구체적 컴퓨터 비전 및 AI 제어 명세

### 3.1 스킨톤 제거 및 마스크 클리핑 (Mask Clipping)
바디 정렬 불일치로 인해 의상 외곽선 밖으로 바디 픽셀이나 배경 노이즈가 번져나가는 현상을 방지하기 위해 강제 클리핑 마스크 연산을 수행합니다.

*   **스킨톤 판정 공식**: MASK_SPEC.md 2.1절의 규칙을 사용해 스킨색 영역의 Alpha를 0으로 강제 소거합니다.
*   **마스크 강제 안착 (Clipping)**: 정형화된 아바타 의상 설계 영역 마스크인 `clothing_mask`를 사용하여, 이 마스크 경계 바깥으로 삐져나온 초안의 모든 픽셀은 **HTML5 Canvas 혹은 Python OpenCV 연산을 통해 물리적으로 지워버려(clipping)** 목선과 어깨선의 정렬 오차를 원천 차단합니다.

### 3.2 도트 리샘플링 및 색상 양자화 (Color Quantization)
축소 시 도트 픽셀들이 지저분하게 뭉개지거나 흐려지는 현상을 막기 위해, 고화질 이미지의 색상 수를 제한하고 경계를 강화하는 **도트 스타일화 알고리즘**을 거칩니다.

1.  **외곽선 강조 (Edge Sharpening)**: 다운스케일 전에 Sobel 또는 Laplacian 필터(혹은 Canvas의 임계치 필터)를 사용하여 의상의 외곽 테두리 아웃라인을 강제로 볼드하게 다듬어 줍니다.
2.  **색상 양자화 (Color Quantization - K-Means)**: 고해상도 원본의 무한한 색상 그라데이션을 도트 특유의 질감으로 바꾸기 위해, 대표색을 **16색 혹은 32색**으로 강제 클러스터링(양자화)하여 중간 톤 노이즈 픽셀을 강제로 단색 블록화합니다.
3.  **정밀 Nearest Neighbor 다운스케일**: 다운스케일 시 주변 픽셀을 흐리게 섞는 보간법을 철저히 금지하고, `Nearest Neighbor` 보간법만을 사용하여 정해진 픽셀 그리드 위에 칼 같은 도트 테두리가 맺히도록 리샘플링합니다.

### 3.3 강력한 ComfyUI 가중치 구속 (Constraint Weighting)
ComfyUI 동작 확장 단계에서 캐릭터의 원래 뼈대 형태가 틀어지지 않도록, KSampler 생성 시 보조 제어 모델의 강도를 극단적으로 강하게 고정합니다.

*   **Canny ControlNet 가중치**: `0.85 ~ 1.0`로 극단적으로 높게 설정하여 캐릭터 의상의 실루엣과 바디 경계선이 AI 맘대로 늘어나거나 줄어들지 않고 고정되도록 강제 구속합니다.
*   **IP-Adapter Strength**: 초안의 의상 테마 반영 가중치는 `0.6 ~ 0.7` 정도로 조율하여, Stand1의 디자인 디테일을 가져오되, 모션 변형(걷거나 뛰는 프레임의 치마 들림 등)이 자연스럽게 연출되도록 유연성을 부여합니다.
