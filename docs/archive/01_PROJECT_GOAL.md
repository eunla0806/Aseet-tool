# 프로젝트 목표

## 한 줄 목표

MapleStory Worlds 기본 바디 템플릿 124개 동작에  
일관된 의상을 자동으로 입히는 웹 도구를 만든다.

---

## 왜 이 앱을 만드는가?

메이플스토리 월드 기본 바디 템플릿에는 여러 동작 프레임이 있다.

예:

- stand
- walk
- jump
- sit
- rope
- ladder
- attack
- hit
- prone

목표는 이 프레임들에 같은 의상을 자연스럽게 입히는 것이다.

하지만 AI에게 124개 프레임을 각각 따로 생성하게 하면 문제가 생긴다.

---

## 핵심 문제

AI가 프레임별로 이미지를 새로 생성하면 다음 문제가 발생할 수 있다.

- 프레임마다 의상 색이 달라짐
- 단추, 장식, 소매 길이가 달라짐
- 몸 포즈가 미세하게 바뀜
- walk 루프에서 옷이 흔들림
- ladder / rope 동작에서 팔과 옷이 어긋남
- attack 동작에서 소매나 하의가 깨짐
- 살이 비치거나 빈틈이 생김
- 투명 배경이나 픽셀 품질이 일정하지 않음

그래서 이 프로젝트는 AI에게 124개 프레임을 각각 새로 그리게 하지 않는다.

---

## 해결 방향

이 프로젝트의 핵심 방식은 다음과 같다.

```text
AI 또는 사용자가 의상 파츠를 만든다.
앱이 같은 파츠를 124개 프레임에 자동으로 배치한다.
어긋남과 살 비침은 숫자 조절값으로 보정한다.
```

즉, AI는 의상 전체를 매번 새로 그리는 역할이 아니다.

AI는 다음 정도만 담당한다.

- 몸통 의상 파츠 생성
- 소매 파츠 생성
- 하의 또는 코트 하단 파츠 생성
- 후면 장식 파츠 생성

앱은 다음을 담당한다.

- 파츠 업로드
- 현재 프레임 위에 파츠 겹쳐보기
- 위치 조절
- 크기 조절
- 빈틈 보정
- 액션별 조절값 저장
- 프레임별 조절값 저장
- 살 비침 경고
- ZIP 내보내기

---

## 현재 앱에 이미 있는 기능

이 프로젝트는 빈 프로젝트가 아니다.

현재 앱에는 이미 다음 기능과 데이터가 있다.

- 124프레임 동작 카탈로그
- 3개 완성본 샘플 의상 데이터
- 샘플 의상 레이어 미리보기
- 전면 / 후면 의상 이미지 업로드
- 프레임별 위치 보정
- 샘플 비교 기능
- 레이어 표시 / 숨기기 기능
- ZIP 내보내기
- project_config.json 생성 구조

따라서 새 기능은 기존 앱을 갈아엎는 방식으로 만들지 않는다.

새 기능은 기존 앱 위에 “매크로 파츠 리깅 기능”을 추가하는 방식으로 만든다.

중요:

- 기존 샘플 비교 기능을 유지한다.
- 기존 전면 / 후면 업로드 기능을 유지한다.
- 기존 ZIP export 기능을 유지한다.
- 새로 추가하는 torso / sleeve / lower / back 파츠 기능은 기존 기능과 충돌하지 않게 만든다.

---

## 중요한 원칙

### 1. AI가 124개 프레임을 각각 새로 그리지 않는다

나쁜 방향:

```text
AI에게 stand1_0 생성 요청
AI에게 stand1_1 생성 요청
AI에게 walk1_0 생성 요청
AI에게 walk1_1 생성 요청
...
```

이 방식은 프레임마다 의상이 달라질 가능성이 높다.

좋은 방향:

```text
AI 또는 사용자가 torso 파츠 생성
AI 또는 사용자가 sleeve 파츠 생성
AI 또는 사용자가 lower 파츠 생성
앱이 같은 파츠를 124프레임에 반복 적용
```

---

### 2. 같은 파츠를 재사용한다

의상 일관성은 AI가 아니라 앱의 구조로 유지한다.

예:

```text
torso.png  → 여러 프레임에서 재사용
sleeve.png → 여러 프레임에서 재사용
lower.png  → 여러 프레임에서 재사용
back.png   → 필요한 프레임에서 재사용
```

같은 이미지를 재사용하면 색상, 장식, 형태가 프레임마다 바뀌는 문제를 줄일 수 있다.

---

### 3. 수동 픽셀 보정보다 숫자 조절을 우선한다

이 프로젝트의 목표는 수동 픽셀 보정을 줄이는 것이다.

따라서 문제를 발견했을 때 포토샵으로 한 프레임씩 고치는 방식은 우선하지 않는다.

대신 아래처럼 숫자로 조절한다.

```text
torso offsetX +1
sleeve offsetY -2
lower scale 1.05
walk1 액션에서 sleeve offsetX +1
ladder 액션에서 torso offsetY -1
walk1_2 프레임만 lower offsetY +2
```

이런 방식이면 같은 문제를 여러 프레임에 반복 적용할 수 있다.

---

## 의상 파츠 기본 구조

처음 버전에서는 너무 세밀한 파츠 분해를 하지 않는다.

기본 파츠는 4개만 사용한다.

| 파츠 | 설명 |
|---|---|
| torso | 몸통 의상 |
| sleeve | 소매 |
| lower | 하의, 치마, 코트 하단 |
| back | 망토, 등 장식, 후면 장식 |

이 정도를 “매크로 파츠”라고 부른다.

---

## 처음부터 하지 않을 것

초기 버전에서는 아래 기능을 하지 않는다.

- 완전 자동 AI 의상 생성
- 124개 프레임 일괄 AI 생성
- 완전 파츠 분해
- 뼈대 기반 리깅
- 복잡한 마스크 자동 생성
- Photoshop PSD 자동 생성
- 실제 AI API 연결

먼저 단순하고 확실한 기능부터 만든다.

---

## 첫 번째 목표

첫 번째 목표는 다음이다.

```text
torso / sleeve / lower / back 이미지를 업로드한다.
현재 바디 프레임 위에 겹쳐본다.
위치와 크기를 숫자로 조절한다.
액션별 / 프레임별 조절값을 저장한다.
살 비침 가능성을 경고한다.
```

이 기능이 안정적으로 동작한 뒤 AI 생성 기능을 추가한다.

---

## Codex 작업 원칙

Codex는 한 번에 전체 앱을 만들지 않는다.

항상 작은 Step 단위로 작업한다.

Codex는 매 작업마다 다음을 지켜야 한다.

1. 이번 Step만 진행한다.
2. 다음 Step은 진행하지 않는다.
3. 작업 전 수정할 파일을 먼저 설명한다.
4. 기존 기능과 UI를 최대한 유지한다.
5. 초보자가 이해할 수 있게 설명한다.
6. 가능하면 한국어 주석을 추가한다.
7. 실제 AI API 연결은 마지막 단계에서 한다.
8. API 키를 프론트엔드에 넣지 않는다.

---

## 최종 방향

최종 앱은 다음 흐름을 목표로 한다.

```text
1. body template 124프레임 로드
2. 의상 파츠 업로드 또는 AI 생성
3. 현재 프레임 위에 파츠 미리보기
4. global 조절값 적용
5. action 조절값 적용
6. frame 조절값 적용
7. 살 비침 / 어긋남 경고
8. 문제 프레임만 조절
9. ZIP export
10. MSW 제작 과정에 활용
```

이 프로젝트의 핵심은 AI 생성 자체가 아니라  
AI 결과물을 124프레임에 일관되게 적용하는 제작 도구를 만드는 것이다.
