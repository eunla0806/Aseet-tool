## 현재 작업 목표

`torso / sleeve / lower / back` 파츠 이미지를 업로드하고,  
업로드한 이미지를 간단히 미리볼 수 있는 패널을 만든다.

이번 작업은 “파츠 업로드 UI”를 만드는 단계다.

아직 `PreviewStage` 위에 파츠를 합성하지 않는다.  
아직 위치 조절 기능도 만들지 않는다.  
아직 AI 기능도 만들지 않는다.

## 프로젝트 핵심 방향

이 프로젝트는 MapleStory Worlds 기본 바디 템플릿 124프레임에  
일관된 의상을 자동으로 입히는 웹 도구를 만드는 것이 목표다.

중요한 점:

```text
AI가 124개 프레임을 각각 새로 그리는 방식이 아니다.
torso / sleeve / lower / back 같은 매크로 파츠를 만든다.
앱이 같은 파츠를 124프레임에 자동 배치한다.
어긋남은 offset, scale, expandPixels 같은 숫자로 조절한다.
```

---

## 이번 작업에서 다룰 파츠 종류

파츠는 4개만 사용한다.

| 파츠 이름 | 표시 이름 | 설명 | 필수 여부 |
|---|---|---|---|
| torso | 몸통 | 상의, 갑옷, 코트 상단 | 필수 |
| sleeve | 소매 | 팔과 소매 | 필수 |
| lower | 하의/하단 | 바지, 치마, 코트 하단 | 선택 |
| back | 후면 장식 | 망토, 등 장식 | 선택 |

파츠 정보는 Step 2에서 만든 `src/rigging/outfitParts.ts`의 `OUTFIT_PARTS`를 사용한다.

## 이번 작업에서 할 일

이번 작업에서는 아래만 진행한다.

1. `src/components/OutfitPartsPanel.tsx` 파일을 만든다.
2. `OUTFIT_PARTS` 목록을 사용해서 4개 파츠 업로드 영역을 표시한다.
3. 각 파츠마다 PNG 이미지를 업로드할 수 있게 한다.
4. 업로드한 이미지를 작은 미리보기로 보여준다.
5. 필수 파츠와 선택 파츠를 구분해서 표시한다.
6. `App.tsx`에 `OutfitPartsPanel`을 연결한다.
7. 업로드 상태는 `App.tsx`에서 관리하거나, 필요하면 단순한 hook으로 분리한다.

## 업로드 상태 방향

업로드 상태는 단순하게 관리한다.

예상 구조:

```ts
Partial<Record<OutfitPartType, OutfitPartImage>>
```

또는 프로젝트에 더 맞는 간단한 구조를 사용한다.

각 파츠는 다음 정보를 가질 수 있다.

- 파츠 타입
- 파일 객체
- 미리보기 URL

예:

```ts
{
  torso: {
    type: 'torso',
    file: File,
    previewUrl: 'blob:...'
  }
}
```

단, 실제 구현은 기존 코드 스타일에 맞춰 가장 단순하게 한다.

## UI 요구사항

`OutfitPartsPanel`은 초보자가 이해하기 쉬운 UI로 만든다.

각 파츠 영역에는 다음 정보를 보여준다.

- 파츠 이름
- 설명
- 필수 / 선택 표시
- 파일 선택 버튼
- 업로드된 이미지 미리보기
- 업로드된 파일명
- 필요하면 삭제 또는 초기화 버튼

라벨 예시:

- 몸통
- 소매
- 하의/하단
- 후면 장식

필수 표시는 다음처럼 쉽게 보여준다.

```text
필수
선택
```



## UI 디자인 원칙

이번 Step 3의 UI는 최대한 작고 단순하게 만든다.

`04_DESIGN_DIRECTION.md`의 파스텔톤, 둥근 모서리, 초보자 친화적인 분위기는 참고한다.

하지만 이번 Step에서는 디자인 기능을 과하게 넣지 않는다.

원칙:

- 새 패널은 접을 수 있는 작은 섹션으로 만든다.
- 기본 상태는 접힘 또는 작은 요약 상태를 우선한다.
- 파츠 4개는 큰 카드가 아니라 간단한 목록 형태로 보여준다.
- 각 파츠에는 이름, 필수/선택 표시, 파일 선택 버튼, 작은 미리보기만 둔다.
- 긴 설명문은 항상 보이지 않게 한다.
- 기존 앱의 색상, 여백, 버튼 스타일을 따른다.
- 기존 화면 배치를 크게 바꾸지 않는다.
- Step 3에서는 PreviewStage에 파츠를 합성하지 않는다.
- Step 3에서는 위치 조절 UI를 만들지 않는다.
- Step 3에서는 D-pad, 자동 스냅, 살 비침 경고를 만들지 않는다.

추천 UI:

```text
의상 파츠 업로드 0/4 [펼치기]

몸통        필수   [PNG 선택]   미리보기
소매        필수   [PNG 선택]   미리보기
하의/하단   선택   [PNG 선택]   미리보기
후면 장식   선택   [PNG 선택]   미리보기
```

파츠별 색상은 너무 강하게 쓰지 않는다.

예:

- 몸통: 연한 핑크 포인트
- 소매: 연한 라벤더 포인트
- 하의/하단: 연한 블루 포인트
- 후면 장식: 연한 옐로우 포인트

색상은 테두리나 작은 배지 정도에만 사용한다.
