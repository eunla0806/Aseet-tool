# 🧸 Dot Asset Tool: React 웹 앱 가이드 (APP_GUIDE)

본 문서는 메이플스토리 월드(MapleStory Worlds) 아바타의 의상 에셋 정렬, 리깅, 실시간 모션 검수 및 패키징을 위한 **React 리깅 헬퍼 웹 애플리케이션(`src` 디렉토리)**의 기능과 실행 방법을 정리한 공식 가이드라인입니다.

---

## 1. 개요 (Overview)

본 React 웹 애플리케이션은 저사양 로컬 환경을 고려한 **하이브리드 아세트 파이프라인의 핵심 축**입니다. 
외부 AI(예: Midjourney, Stable Diffusion 등)나 그래픽 편집기에서 제작한 의상 초안 시트를 드래그 앤 드롭으로 로드한 뒤, 아바타 캐릭터 바디 템플릿에 딱 들어맞도록 오프셋 좌표를 리깅하고 실시간 애니메이션으로 피팅 상태를 검수할 수 있는 고기능 GUI 저작 도구입니다.

```txt
[외부 AI 2x2 초안] ➔ [Python 정형화 (43x68)] ➔ [React 리깅 헬퍼 (X/Y 오프셋 정렬 & 애니메이션 검수)] ➔ [최종 MSW 스프라이트 시트]
```

---

## 2. 핵심 기능 상세 (Core Features)

웹 애플리케이션은 사용자 편의성을 위해 직관적인 **4단계 단계별(Step-by-Step) 워크플로우**를 제공합니다.

### 2.1 STEP 1: 캐릭터 모션 프리뷰 및 바디 매핑
*   **실시간 모션 애니메이션**: `src/data/frames.json` 데이터베이스에 구축된 프레임 구조와 연동하여 아바타의 32가지 모든 동작(대기, 걷기, 점프, 앉기, 사다리 타기, 공격 모션 등 총 124프레임)을 끊김 없이 재생 및 정지합니다.
*   **재생 제어**: 애니메이션 재생 속도 조절(`느리게`, `보통`, `빠르게`), 픽셀 단위 격자 가이드(`Grid Overlay`), 캔버스 줌 배율 조절(최대 8배 이상 줌인 지원) 기능이 포함되어 있어 미세 도트 배치 확인에 최적화되어 있습니다.

### 2.2 STEP 2: 외부 초안 업로드 및 사전 정렬 (Upload & Align)
*   **초안 드래그 앤 드롭 업로드**: 로컬 또는 AI로 추출된 앞면(`Front`), 뒷면(`Back`) 투명 배경 의상 시트 이미지를 웹 브라우저 화면에 끌어다 놓는 방식으로 즉시 업로드할 수 있습니다.
*   **비주얼 사전 정렬 (Visual Alignment)**: 마우스 드래그 또는 정밀 조작 슬라이더(Nudger)를 활용하여 의상의 기본 기준 X/Y 오프셋과 전체 스케일(Scale)을 캐릭터 바디에 알맞게 맞춥니다.
*   **의상 파츠 조합 패널 (`OutfitPartsPanel`)**: 상의(mailChest), 하의(pants), 전신옷(mail), 팔/소매(mailArm) 등 메이플 월드 규격에 맞는 파츠 바인딩 상태를 직관적으로 체크할 수 있습니다.

### 2.3 STEP 3: 프레임 단위 미세 오프셋 조율 (Nudge & Calibrate)
*   **프레임 개별 미세 보정**: 움직이는 모션 프레임 중 특정 구간(예: 걷기 프레임 중 하체가 뒤로 빠지는 순간 등)에서 의상이 바디 핏에서 어긋나는 현상을 방지하기 위해, **각 프레임 단위로 X/Y 픽셀 좌표를 미세 쉬프트 이동**하여 고정합니다.
*   **세부 레이어 필터 토글**: 몸통, 하체, 팔, 겉옷 등 바디와 옷의 미세 레이어 가시성(`Visibility`)을 개별 토글할 수 있어, 겹치는 복잡한 파츠 뒤의 도트 안착 상태를 손쉽게 추적합니다.
*   **정답 의상 반투명 대조군 비교 (`Overlay Guide`)**: `05_멜로디 소녀` 등 검증된 정답 레이어(`clothing_only_source`)를 캔버스 배경에 반투명 오버레이하거나 깜빡이게(Toggle) 연동하여, 제작 중인 의상의 물리적 핏과 아웃라인의 정합성을 1:1로 비교 검수할 수 있습니다.

### 2.4 STEP 4: 프로젝트 설정 내보내기 & 패키징 (Export)
*   **프로젝트 설정 저장/로드**: 픽셀 단위로 공들여 조율한 전체 프레임 오프셋 좌표 및 레이어 가시성 정보를 `ProjectConfig` 규격 파일(`.json`)로 즉시 컴퓨터에 저장하거나 다시 불러와 이어서 작업할 수 있습니다.
*   **에셋 패키징 내보내기**: 메이플스토리 월드 개발자 센터에 스프라이트 시트로 즉시 등록하여 사용할 수 있도록 최적화된 리소스를 내보냅니다.

## 3. 로컬 실행 및 사용 방법 (How to Run)

React 앱은 개발/빌드 도구인 **Vite**를 채택하여 즉각적으로 서버를 시작하고 실시간 핫 리로딩(HMR)을 지원합니다.

### 3.1 사전 요구사항
*   컴퓨터에 **Node.js** (LTS v18 이상 권장)가 설치되어 있어야 합니다.

### 3.2 패키지 의존성 설치 (필요시)
이미 `src` 내에 필수 패키지 라이브러리(`node_modules`)가 사전에 다운로드 및 압축 해제되어 포함되어 있으므로 보통 이 단계는 건너뛰어도 무방합니다. 만약 의존성 문제나 누락이 감지되면 폴더로 이동 후 아래 명령어를 수행합니다.
```bash
# 터미널을 열고 src 경로로 이동
cd "E:\Aseet tool\src"

# 의존성 설치 명령어 실행
npm install
```

### 3.3 로컬 개발 서버 시작 (Dev Server 실행)
터미널에서 아래 명령을 실행하여 개발 서버를 켭니다.
```bash
npm run dev
```

서버 구동 성공 시 아래와 같이 터미널에 주소가 출력됩니다.
```txt
  VITE v8.0.12  ready in 184 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 3.4 웹 브라우저 접속
인터넷 익스플로러를 제외한 최신 브라우저(Chrome, Edge, Safari 등)를 열어 **`http://localhost:5173/`** 주소로 접속하면, 세련된 아바타 리깅 웹 뷰어를 즉시 사용할 수 있습니다.

## 4. 디렉토리 및 구성 파일 구조

`src` 내부 폴더 구조는 깔끔하게 도메인별로 캡슐화되어 있습니다.

```txt
src/
├── components/          # UI 조작용 컴포넌트 목록
│   ├── AppHeader.tsx        # 로고 및 마스터 헤더
│   ├── FooterNavigation.tsx # 마법사 Step 전환 버튼
│   ├── MotionControls.tsx   # 재생속도, 모션종류, 줌 등 플레이어 컨트롤러
│   ├── OutfitPartsPanel.tsx # 의상 파츠 업로드 및 스펙 관리
│   ├── PreviewStage.tsx     # 실시간 캐릭터 렌더링 캔버스 영역
│   └── StepProgress.tsx     # 마법사 단계별 인디케이터
├── data/                # 데이터 베이스
│   ├── frames.json          # 바디 템플릿 프레임 정보 및 Y/X 좌표 원본
│   └── layers.ts            # 레이어 그룹 및 역할(Role) 매핑
├── domain/              # 비즈니스 도메인 핵심 로직
│   ├── frameOffsetGroups.ts # 오프셋 대량 보정용 프레임 그룹화 처리
│   ├── projectConfig.ts     # 리깅 프로젝트 불러오기/내보내기 파서 및 직렬화
│   └── samplePresets.ts     # 멜로디 소녀 등 가이드 비교용 샘플 프리셋
├── rigging/             # 리깅 엔진 및 레이어 매핑
│   └── outfitParts.ts       # 의상 파츠 타입(상의, 하의 등)별 규격 정의
├── App.tsx              # 앱의 마스터 상태 및 워크플로우를 통제하는 메인 진입점
├── index.css            # 고품질의 세련된 UI 스타일링(CSS) 정의서
└── main.tsx             # React 가상 DOM 렌더러 진입 스크립트
```

## 5. package.json의 유용한 부가 스크립트

개발 서버 구동(`npm run dev`) 이외에도, `package.json`에는 검수 및 자동화를 위한 다양한 유틸리티 스크립트가 준비되어 있습니다.

*   `npm run build`: React 앱을 최적화된 프로덕션 번들 파일로 컴파일하고 빌드합니다. (최종 배포용)
*   `npm run verify:psd`: MSW PSD 템플릿의 규격 및 오프셋 정합성 검사를 검증합니다.
*   `npm run verify:project-config`: 리깅 프로젝트 JSON 설정 파일이 스키마 규격 계약을 통과하는지 유효성을 검사합니다.
*   `npm run sample:psd`: 리깅 테스트를 위한 테스트용 가상 PSD 구조를 합성 생성합니다.
